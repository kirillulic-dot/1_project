---

## 🗓 **Учебный план на 8 месяцев (C/C++ + Python + Git + RTOS + базы)**

---

### 🩵 **Месяц 1 — Основа языка C и системного мышления**

**Цель:** закрепить синтаксис, работу с памятью и файлами.
**Ты уже многое знаешь — поэтому часть пройдёшь быстрее.**

📘 **Темы:**

* Структура программы на C (`main`, функции, заголовочные файлы);
* Типы данных, указатели, массивы, строки;
* `malloc/free`, динамические массивы;
* Работа с файлами (`fopen`, `fprintf`, `fscanf`, `fseek`, `ftell`);
* Разделение кода на `.h` и `.c` файлы.

🧠 **Проект:**

> Написать менеджер базы данных «Города и страны» — с меню, добавлением, поиском, сохранением в файл.
> (можно доработать твой текущий проект, добавить проверку ввода, сортировку, удаление и чтение структур из файла)

---

### 💙 **Месяц 2 — Углублённый C и основы C++**

**Цель:** сделать код модульным и понятным, добавить базовые структуры C++.

📘 **Темы:**

* Структуры и вложенные структуры;
* Обработка ошибок (коды возврата, `errno`);
* Основы C++ (классы, конструкторы, инкапсуляция);
* Разбор Makefile или компиляции через `gcc`/`g++`.

🧠 **Проект:**

> «Менеджер студентов» — программа на C++, где можно добавлять студентов, считать средний балл, сохранять в файл, использовать классы.

---

### 💚 **Месяц 3 — Python для автоматизации и тестов**

**Цель:** освоить Python для скриптов и тестирования программ.

📘 **Темы:**

* Основы Python (типизация, функции, списки, словари);
* Работа с файлами (`open`, `read`, `write`);
* Подключение внешних библиотек (`os`, `subprocess`, `pytest`);
* Основы тестирования: как писать простые юнит-тесты.

🧠 **Проект:**

> Python-скрипт, который запускает твою программу на C, подаёт тестовые данные и проверяет результаты (автотестер).

---

### 💛 **Месяц 4 — Git и GitLab CI/CD**

**Цель:** научиться работать как настоящий разработчик.

📘 **Темы:**

* Основы Git (init, clone, branch, merge, commit, push);
* GitHub / GitLab;
* CI/CD — что это, как автоматизировать сборку.

🧠 **Проект:**

> Создать GitHub-репозиторий с твоим проектом, настроить простейший `.gitlab-ci.yml` или GitHub Actions для автоматической сборки C-проекта.

---

### 💜 **Месяц 5 — Работа с базами данных**

**Цель:** понять основы хранения данных (SQL и NoSQL).

📘 **Темы:**

* SQL: таблицы, запросы `SELECT`, `INSERT`, `UPDATE`, `DELETE`;
* MongoDB: коллекции, документы, базовые CRUD-операции;
* Подключение Python к базе данных (`sqlite3` или `pymongo`).

🧠 **Проект:**

> Программа для хранения данных о пользователях с сохранением в базу (например, SQLite).
> Потом сделать Python-скрипт, который считывает эти данные и формирует отчёт.

---

### ❤️ **Месяц 6 — Основы RTOS и embedded**

**Цель:** понять принципы многозадачности и управления устройствами.

📘 **Темы:**

* Что такое RTOS, зачем она нужна;
* Задачи, прерывания, таймеры;
* Знакомство с FreeRTOS (через симулятор или STM32CubeIDE);
* UART, SPI, GPIO — ввод/вывод микроконтроллера.

🧠 **Проект:**

> Написать эмуляцию RTOS: программа с “псевдопотоками” (например, чтение данных + логирование + анализ).

---

### 🧡 **Месяц 7 — Мини-проект и интеграция всего**

**Цель:** объединить всё, чему научился.

🧠 **Проект:**

> «Система мониторинга датчика»

* C-программа собирает данные (имитация датчика),
* пишет в файл,
* Python-скрипт читает файл и визуализирует график,
* Git отслеживает версии,
* база данных хранит результаты.

---

### 💜 **Месяц 8 — Подготовка к трудоустройству / стажировке**

**Цель:** оформить портфолио и подготовиться к собеседованиям.

📘 **Действия:**

* Создай 3–4 проекта на GitHub;
* Напиши README с описанием;
* Пройди тестовые задания с hh.ru или habr.career;
* Изучи вопросы по C/C++, Python, Git, и немного по RTOS;
* Подготовь краткое резюме на английском.

---

## 🧭 Что будет в итоге

Через 8 месяцев ты:

* умеешь писать программы на **C/C++ и Python**,
* знаешь **базы данных, Git и CI/CD**,
* понимаешь **основы RTOS и embedded**,
* имеешь 3–5 реальных проектов в портфолио,
* можешь претендовать на позицию **Junior Developer / Embedded Developer / QA Automation**.
